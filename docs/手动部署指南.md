# 手动部署指南 — hy-platform-web

## 前置准备

| 项目 | 值 |
|------|-----|
| 服务器 IP | `124.221.127.123` |
| 服务器用户 | `root` |
| 服务器部署目录 | `/opt/hy-platform-web` |
| Harbor 仓库地址 | `hbu.docker` |
| Harbor 项目 | `hy-platform` |
| 镜像名 | `hbu.docker/hy-platform/hy-admin` |
| 服务器访问端口 | `80`（前端页面） |
| API 网关端口 | `8080`（后端 gateway，暂未部署） |

---

## 第一步：本地构建镜像

在 `react-monorepo-admin` 项目根目录执行：

**CMD / Git Bash：**

```bash
docker build --build-arg VITE_APP_BASE_URL=http://124.221.127.123:8080/api --build-arg VITE_APP_TITLE=呼呼呼 -t hbu.docker/hy-platform/hy-admin:latest .
```

**PowerShell：**

```powershell
docker build `
  --build-arg VITE_APP_BASE_URL=http://124.221.127.123:8080/api `
  --build-arg VITE_APP_TITLE=呼呼呼 `
  -t hbu.docker/hy-platform/hy-admin:latest .
```

> `VITE_APP_BASE_URL` 是浏览器中前端调用后端 API 的地址。后端部署后如需修改，改这里的值重新构建即可。

构建成功后验证：

```bash
docker images hbu.docker/hy-platform/hy-admin
```

---

## 第二步：登录 Harbor 并推送镜像

```bash
# 登录 Harbor
docker login hbu.docker

# 推送镜像
docker push hbu.docker/hy-platform/hy-admin:latest
```

---

## 第三步：把部署文件上传到服务器

```bash
# 创建部署目录
ssh root@124.221.127.123 "mkdir -p /opt/hy-platform-web"

# 上传 compose 文件
scp docker-compose.prod.yml root@124.221.127.123:/opt/hy-platform-web/docker-compose.yml
```

> 上传后文件名改为 `docker-compose.yml`，这样服务器上直接用 `docker compose up -d` 即可，无需指定 `-f`。

### docker-compose.prod.yml 内容检查

上传前确认文件中的这几个值：

```yaml
image: ${HARBOR_REGISTRY:-hbu.docker}/hy-platform/hy-admin:${IMAGE_TAG:-latest}
```

如果服务器上不想传环境变量，可以直接写死：

```yaml
image: hbu.docker/hy-platform/hy-admin:latest
```

> ⚠️ 已在文件中的话就无需修改。

---

## 第四步：服务器上拉取镜像并启动

SSH 到服务器：

```bash
ssh root@124.221.127.123
```

在服务器上执行：

```bash
# 进入部署目录
cd /opt/hy-platform-web

# 登录 Harbor
docker login hbu.docker

# 拉取镜像
docker pull hbu.docker/hy-platform/hy-admin:latest

# 启动容器
docker compose up -d

# 查看运行状态
docker ps --filter name=hy-platform-web

# 验证健康检查
curl http://localhost/health
```

http://124.221.127.123/

应能看到前端页面正常加载。

## 常用运维命令

```bash
# 查看日志
docker logs -f hy-platform-web

# 查看最近 100 行日志
docker logs --tail 100 hy-platform-web

# 重启容器
docker compose -f /opt/hy-platform-web/docker-compose.yml restart

# 停止容器
docker compose -f /opt/hy-platform-web/docker-compose.yml down

# 更新到新版本
cd /opt/hy-platform-web
docker compose down
docker pull hbu.docker/hy-platform/hy-admin:latest
docker compose up -d
docker ps --filter name=hy-platform-web
```

## 防火墙检查

如果浏览器访问不了，检查服务器防火墙是否开放 80 端口：

```bash
# 查看防火墙状态
firewall-cmd --list-ports    # firewalld
# 或
ufw status                    # ufw
# 或
iptables -L -n                # iptables

# 开放 80 端口
firewall-cmd --add-port=80/tcp --permanent && firewall-cmd --reload   # firewalld
# 或
ufw allow 80/tcp                                                       # ufw
```

如果用的是云服务器（阿里云/腾讯云等），还需要在**安全组**中放行 80 端口。

## 注意事项

1. **后端未部署**，页面能正常访问但 API 调用会失败（不影响看页面效果）
2. `VITE_APP_BASE_URL` 是**构建时变量**，修改后需重新构建和推送镜像
3. 镜像推送需要 Harbor 登录状态，如果过期需重新 `docker login hbu.docker`
4. 后端部署后需确保 `gateway` 容器在 `hu-platform-net` 网络中运行，Nginx 才能通过容器名 `gateway` 代理到后端
